SQL转义字符

场景

为了解决字符串中单引号的问题，所以出现了转义字符。
常见的转义字符有“’” 或 “、” （加了双引号“”）
插入值中带转义字符，如单引号‘ 或` 或\

INSERT INTO `account` VALUES (1, 1000, 'keith');
INSERT INTO `account` VALUES (2, 1000, '\'keith');
INSERT INTO `account` VALUES (3, 1000, '`keith');
INSERT INTO `account` VALUES (4, 1000, '\\keith');

其中每一条语句第三个值分别插入值为 keith、’keith、`keith、\keith 的数据
相应的，查询语句中如果出现转义字符，也用类似的方法查询即可。

select * from `account` where name = 'keith';
select * from `account` where name = '\'keith';
select * from `account` where name = '`keith';
select * from `account` where name = '\\keith';

转义字符\，可以使用`可以查出`的数据

select * from `account` where name = '\`keith';

‘和\两个转义字符使用，会因为先匹配了’’，导致\keith’成为一个独立的sql语句，故报错。

select * from `account` where name = ''\keith';

转义字符’，使用’’ 和'都可以查出’的数据

select * from `account` where name = '''keith';
select * from `account` where name = '\'keith';

其中如果变成

select * from `account` where name = ''keith';

由于SQL中单引号表示字符串的开始和结束字符，所以SQL编译器会默认以为查询的为’’，导致后面的keithi’是一条独立的错误sql语句，故报错。
